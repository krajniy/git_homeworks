Зачем нужен GIT и какие задачи решают с его помощью?
	GIT - система контроля версий, которая позволяет решать следующие задачи:
		- отслеживать изменения проекта;
		- откатываться изменения и возвращаться к прошлым версиям проекта;
		- работать над проектом параллельно с другими членами команды;
		- продолжить работу с кодом с любого устройства, если код хранится в удаленном репозитории;
		- обеспечивает безопасность, т.к. не доступен для модификаций извне (благодаря шифрованию).


Что такое репозиторий GIT и где он находится?
		Репозиторий - "папка"). Каталог файловой системы, хранится локально на компьютере, а также удаленно, если разместить его, например, на GitHub.


Для чего используется файл .gitignore?
		В файле .gitignore указываются файлы, которые надо проигнорировать при коммите и не добавлять в репозиторий.  Например, чтобы не добавлять скомпилированные классы в джаве, надо указать *.class.

Что такое "коммит"?
		Команда commit фиксирует изменения в репозитории. Разница между новой версией и прошлой.


Укажите базовые команды в рамках работы с GIT:

Настройка аккаунта локального пользователя:
		git config --global user.name "Elena Ivanishcheva"
		git config --global user.email "krajniy@gmail.com"

Создание репозитория:
		mkdir git-project
		cd git-project		

Добавление файлов в индекс и выполнение коммита.
		git add . (или git add somefile)
		git commit (или git commit -m "Initial commit")

Просмотр журнала коммитов.
		git log
